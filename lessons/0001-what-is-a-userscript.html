Copyability · Lesson 0001

什么是 Tampermonkey 用户脚本？

一句话：Tampermonkey 是装在 Chrome 里的“脚本管理器”，用户脚本是由它按规则注入网页并执行的一小段 JavaScript。

ChromeTampermonkey用户脚本匹配的网页

两个东西，不要混为一谈

Tampermonkey

浏览器扩展。负责保存、启停、匹配和执行用户脚本，也提供一些额外 API。

用户脚本

你自己编写或安装的 JavaScript 文件。它声明自己在哪些网址运行，然后读取或修改页面。

用户脚本顶部有一个“元数据块”。它不是业务代码，而是给 Tampermonkey 的运行说明：

// ==UserScript==
// @name         Copyability prototype
// @match        https://example.com/docs/*
// @grant        none
// ==/UserScript==

console.log('只在匹配的文档页面运行');
字段含义
@name脚本在管理器中显示的名称。
@match允许脚本运行的网址范围；应尽量收窄。
@grant脚本申请的 Tampermonkey 特权 API；没有需要时使用 none。

为什么适合做首版

完整 Chrome 扩展通常还需要清单、权限声明、后台逻辑和打包结构。用户脚本把这些脚手架压缩成一个文件，因此适合快速回答：“浏览器页面里能不能可靠实现这个交互？”

它不是魔法：只能在被允许的页面环境中工作，页面结构变化也可能让脚本失效。若最终需要发布、自动更新、复杂界面或多页面协作，再迁移成 Chrome 扩展更合适。

与 Copyability 的关系

首版可以将运行范围限制在目标文档域名，只处理授权浏览者已经看见并选择的文本。这样能用最少工程量验证核心行为，同时保持权限范围清晰。
