Copyability · Reference

运行流程速查

从一次用户复制请求，到纯文本进入系统剪贴板。

识别快捷键验证 DOM Range触发 copy确认后拦截

元数据边界

@match仅匹配 https://my.feishu.cn/wiki/*
@run-atdocument-start，尽早注册事件监听器
@grantnone，不申请 GM_* 特权 API

事件接力

阶段关键动作退出条件
keydown识别 Cmd/Ctrl+C，读取 Selection 和 Range错误按键、空选区
验证检查端点、相交块、全部文本节点包含不支持内容
准备selection.toString() → pendingPlainText空白文本、提取异常
触发document.execCommand("copy")命令未触发 copy
copy写入 text/plain，设置成功标记缺少 clipboardData
收尾清空 pending；成功才阻止宿主事件失败则显示短暂提示

DOM 白名单

可成为选区内容：text、heading1、heading2、heading3、bullet。

仅作为透明祖先：page、callout。

明确不处理：表格、代码块、评论、白板、嵌入应用、图片内容及其他未知块。

三个状态变量

pendingPlainText两个事件之间的一次性交接数据
copySucceeded决定是否可以安全拦截飞书后续事件
failureTimer管理唯一的短暂错误提示

最值得记住的风险

document.execCommand("copy") 已弃用。当前实现依赖它在用户按键期间触发可写的 copy 事件，因此每次 Feishu 或 Chrome 变化后都应重新跑自动化与真实页面验收。

原始资料：Tampermonkey Documentation、Selection、copy event、execCommand()。
